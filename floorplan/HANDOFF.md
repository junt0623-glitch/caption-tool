# 展覧会図面作成アプリ — 引き継ぎメモ

## 現状
`index.html` 単一ファイルの展覧会平面図エディタ(SVGベース、外部依存なし)。
チャットのClaudeが破損ファイルから再構築し、レイヤー・在庫・作品リスト連携まで拡張した版。

## レイヤー構成(描画順)
1. **最背面: 展示室図面** — 4面をデフォルト登録(展示室1-A（旧第1）/1-A（旧第2）/1-B/2-B)。
   何も選択していないときの右パネルで選択し、図面画像(PNG/JPG。PDFは要変換)を
   読み込み・倍率/位置/透明度を調整。A3の範囲内に最大サイズで自動フィット。
   画像は state.rooms.list[].img に dataURL で保存され、JSON保存に含まれる
2. 壁
3. **ケース・展示台(在庫方式)** — 印刷範囲の外に種類ごと1つのパレットを表示。
   可動ケース9種(α×2, β×10, 180角×2, 横長免震×5, 横長免震背景×1,
   横長非免震×1, 横長非免震背景×2, 行燈免震×5, のぞき×5)は左側に縦一列、
   展示台5種(1.53×0.9×18, 1.8×0.9×12, 0.9×0.9×7, 2.0×2.0×2, 1.2×0.9×30)は右側に縦一列。
   パレットをクリック/ドラッグすると1台取り出せて残数が減り、削除すると残数に戻る。
   残数 = total −「stockKeyが一致する配置済みオブジェクト数」で常に導出(同期バグ防止)
4. **作品** — 図面上は「番号」バッジのみ表示(作品名等は出さない)。
   作品リストの番号に対応する画像があればそれを表示
5. **最前面: テキスト**

各レイヤーは右パネル上部のチェックボックスで表示/非表示を切替(ビュー状態、保存対象外)。
描画順は type から layerOf() で決まり、同レイヤー内は objects 配列順(安定ソート)。

## 作品リスト(CSV)と画像の連携
- ヘッダーの「📋 作品リスト」でCSV/TSVを読み込む(Excelから「CSV形式で保存」を想定)。
  「番号」列(なければ1列目)・「作品名」「作家名」列を自動判別。
  Shift_JIS/UTF-8両対応(UTF-8で化けたらShift_JISで再デコード)
- 作品ツールでの配置時、リストの未配置の先頭番号を自動割り当て。番号は右パネルで変更可
- 「🖼 作品画像」で画像を一括読み込み。ファイル名の数字(例: 10001.jpg)を
  normNo() で正規化してリストの「番号」と対応付ける。
  長辺512px・JPEG品質0.72に圧縮して state.images[番号] に保存(JSON保存に含む)
- 番号の照合は normNo()(全角→半角、数字抽出、先頭ゼロ除去)で統一すること

## 主な機能
- 壁の描画(ドラッグ、直交スナップ / Shiftで自由角度)
- 壁の連結(既存の壁の端点400mm以内に近づけると吸着。連結時は直交スナップより優先)
- Ctrl+Zで元に戻す / Ctrl+Y・Ctrl+Shift+Zでやり直し(履歴50件)。
  ※画像(state.images / rooms.img / objects.img)は履歴スナップショットから除外し、
  Undo/Redoをまたいで現在値を維持する(メモリ対策。snapshot()/applySnapshot()参照)
- タッチ2本指のピンチズーム・パン
- 用紙はA3横(既定)/A3縦のみ。SHEET = A3実寸×100(横42000×29700mm)
- **縮尺(1〜100%)**: ケース・展示台の図面に対する表示サイズ倍率(state.scalePct)。
  背景図面の縮尺が不明でも、この%を調整してケースと図面の大きさを合わせる。
  実寸mm(o.w/o.h)は変えず描画時に fsc() を掛けるだけ
- 表題欄は廃止(図面上に展覧会名等は表示しない。state.metaはファイル名用に残存)
- JSON保存/読み込み(旧形式と互換)、PNG書き出し、印刷

## 座標系・単位
- 内部座標はmm。SHEETはA3実寸(mm)×100(state.sheet.presetでa3l/a3p切替)
- グリッドは500mm、壁厚は固定150mm
- viewBoxで拡縮しているズーム/パン

## 既知の設計上の注意点(SVG特有)
- SVGはCSSがプレゼンテーション属性より優先される。font-size等はCSSクラスに書かず
  属性 or インラインstyleで指定すること(過去に文字が極小になるバグがあった)
- PNG書き出し時、外部<style>はcloneNode後にシリアライズされないため
  書き出し関数内でスタイルをSVGに埋め込んでいる(pngBtnのonclick内)

## 未着手・次にやると良さそうなこと
- 展示室図面のPDF直接読み込み(現状は画像化が必要。pdf.js等の導入判断が要る)
- Excel(.xlsx)の直接読み込み(現状はCSV。SheetJS等の導入判断が要る)
- 画像読み込みはUndo対象外(履歴から除外しているため)。必要なら別途復元手段を検討
- 壁の角のマイター(留め継ぎ)処理。連結は端点吸着のみで、鋭角の交差部は
  linecap:squareの重なりで見せている
- 壁の途中(端点以外)への吸着・分割
- 印刷時の等倍保証(現状はブラウザの印刷設定で倍率100%にする前提)
